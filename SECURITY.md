# Security Policy

## Supported versions

Only the latest published minor line is considered supported for security and reliability fixes.

## Reporting a vulnerability

Please report suspected vulnerabilities privately before opening a public issue.

Until a dedicated security contact is published, open a GitHub issue only for non-sensitive security questions and avoid disclosing exploit details publicly.

When reporting a vulnerability, include:

- affected package and version
- reproduction steps
- expected impact
- any suggested mitigations
